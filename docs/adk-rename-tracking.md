# Remaining `agentstack` → `adk` renames (blocked on other apps)

- **Helm chart** — Helm release name is `agentstack` in all deployment docs (`helm upgrade --install agentstack`,
  `helm get values agentstack`, etc.), which drives Kubernetes service names (`svc/agentstack-svc`,
  `agentstack-keycloak`). Rename requires coordinated Helm chart update.
- **Keycloak** — realm name (`realms/agentstack`), roles (`agentstack-admin`, `agentstack-developer`), and client IDs
  referenced in deployment docs. Rename requires Keycloak provisioning update.
- **adk-server** — Python class names drive the generated `schema.d.ts` in `adk-ui`. Rename triggers TypeScript schema
  regeneration.
- **adk-server** — `agentstack://` platform file URI scheme is generated by the server and consumed by `adk-ui`. Rename
  requires coordinated server + UI update.
- **i-am-bee/agentstack-starter** — external starter template repo referenced in deploy-agents docs. Rename depends on
  that repo being migrated.
