Once you've wrapped your agent with the Kagenti ADK server, you need to containerize it and deploy it, so that Kagenti
ADK can run it as a managed service.

## Prerequisites

- Kagenti ADK installed ([Quickstart](/stable/introduction/quickstart))
- Agent wrapped with Kagenti ADK ([Wrap Existing Agents](/stable/deploy-agents/wrap-existing-agents) or
  [Build New Agent](/stable/deploy-agents/building-agents))

## Agent Deploy Process

To deploy your agent to Kagenti ADK, you need to create a github repository with your agent and then deploy from that
repository to Kagenti ADK.

### 1. Create an Agent github repository

You can quickly create an agent repository by using the Kagenti ADK starter template. The
[adk-starter template](https://github.com/kagenti/adk-starter) includes everything you need:

- Production-ready Dockerfile
- GitHub Actions for automated builds
- Kagenti ADK deployment configuration

Clone it and modify it for your agent as follows:

```bash
git clone https://github.com/kagenti/adk-starter my-agent
cd my-agent
# Replace the example agent with your code
```

Alternatively, if you want to start from scratch, you can create your own github repository. You will need to include a
`Dockerfile` with the following information:

```dockerfile
FROM python:3.14-alpine3.23
COPY --from=ghcr.io/astral-sh/uv:0.10.9 /uv /bin/
WORKDIR /app
ADD . .
RUN HOME=/tmp uv sync --no-cache --locked --link-mode copy
ENV PRODUCTION_MODE=true
CMD ["/app/.venv/bin/server"]
```

### 2. Deploy your Agent to Kagenti ADK

Deploying is a two-step process: first build the image and push it to the platform registry, then register it.

**Step 1 — Build and push:**

```bash
kagenti-adk build https://github.com/myorg/my-agent
```

**Step 2 — Register the agent** using the image reference printed by the build command:

```bash
kagenti-adk add registry.cr-system.svc.cluster.local:5000/my-agent-<hash>:latest
```

The `build` command will print the exact `kagenti-adk add` command to run after a successful build.

This workflow automatically:

1. ✓ Builds your Docker image from the GitHub repository
2. ✓ Pushes the image into Kagenti ADK's platform registry
3. ✓ Registers it as an available agent (step 2)

As a specific example, to deploy the adk-starter example agent without modification:

```bash
kagenti-adk build https://github.com/kagenti/adk-starter
# Then run the kagenti-adk add command printed by the build step
```

<Note>
  **Why "push to platform registry"?** Kagenti ADK runs in an isolated VM (Lima on Mac/Linux, WSL on Windows). The
  `build` command builds the image locally with Docker and imports it into the VM's registry so Kagenti ADK can run it.
</Note>

## Github Build Options

The `kagenti-adk build` command accepts any URL that Docker supports as a build context. Supported formats include:

- **Basic URL**: `https://github.com/myorg/myrepo`
- **Git Protocol URL**: `git+https://github.com/myorg/myrepo`
- **URL with .git suffix**: `https://github.com/myorg/myrepo.git`
- **URL with branch or tag**: `https://github.com/myorg/myrepo.git#mybranch`
- **URL with subfolder**: `https://github.com/myorg/myrepo.git#:path/to/agent`

#### Example

To build an agent from a specific branch:

```bash
kagenti-adk build "https://github.com/my-org/my-awesome-agents.git#main"
# Then run the kagenti-adk add command printed above
```

This tells Docker to:

1. Fetch the `main` branch of the `my-awesome-agents` repository.
2. Build the Docker image using the `Dockerfile` found at the repository root (or use `-f` to specify another path).
3. Push the image to the platform registry.

### Verify Deployment

You can check that your agent is registered by running:

```bash
kagenti-adk list
```

Your agent should appear in the list with status information.

## Test Your Agent

Test via CLI:

```bash
kagenti-adk run my-agent "Hello!"
```

Or open the web UI:

```bash
kagenti-adk ui
```

Your agent will be available at `http://adk.localtest.me:8080`

## Advanced Options

### Custom Dockerfile path

If your Dockerfile is not at the repository root, use `-f` to specify its location:

```bash
kagenti-adk build -f path/to/Dockerfile https://github.com/myorg/myrepo
# Then run the kagenti-adk add command printed above
```
