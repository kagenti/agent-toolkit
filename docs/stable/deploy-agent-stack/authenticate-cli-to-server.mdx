title: "Connect CLI to Remote Server"
description: "Connect the Kagenti ADK CLI to your deployed server"
---

Use the Kagenti ADK CLI to configure and manage remote Kagenti ADK deployments. Kagenti ADK follows a client-server architecture, where the CLI acts as a client connecting to a deployed Kagenti ADK server. This allows you to manage agents remotely from your local machine.

## Prerequisites

- Kagenti ADK CLI installed locally ([Quickstart](../introduction/quickstart))
- URL of your deployed Kagenti ADK server
- Authentication credentials for the server

## Connect to a server

Run this command to connect to a remote server:

```bash
kagenti-adk server login
```

The CLI will prompt you for:
- Server URL
- Authentication credentials

Once authenticated, the CLI connects to your specified server and saves the connection for future commands.

## View connected servers

List all servers you've connected to:

```bash
kagenti-adk server list
```

This shows all saved server connections. The active server is marked in the output.

## Switch between servers

To switch to a different connected server:

```bash
kagenti-adk server select
```

Select from your saved connections.

## Disconnect from a server

Remove the currently active server connection:

```bash
kagenti-adk server logout
```

This removes the server from your saved connections. To disconnect from a different server, switch to it first, then logout.
