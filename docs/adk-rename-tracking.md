# Remaining `adk` → `adk` renames (blocked on other apps)

- **Helm chart** — Helm release name is `adk` in all deployment docs (`helm upgrade --install adk`,
  `helm get values adk`, etc.), which drives Kubernetes service names (`svc/adk-svc`,
  `adk-keycloak`). Rename requires coordinated Helm chart update.
- **Keycloak** — realm name (`realms/adk`), roles (`adk-admin`, `adk-developer`), and client IDs
  referenced in deployment docs. Rename requires Keycloak provisioning update.
- **adk-server** — Python class names drive the generated `schema.d.ts` in `adk-ui`. Rename triggers TypeScript schema
  regeneration.
- **adk-server** — `adk://` platform file URI scheme is generated by the server and consumed by `adk-ui`. Rename
  requires coordinated server + UI update.
