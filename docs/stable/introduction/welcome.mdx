Kagenti ADK (Agent Development Kit) takes agents built with any framework or custom code and turns them into [A2A](https://a2a-protocol.org/)-compliant services — with the CLI, SDKs, runtime services, and UI you need to go from agent logic to a working service.

## What's in the kit

- **CLI** — scaffold projects, run agents locally, and deploy to Kagenti
- **Python SDK** — wrap your agent with A2A, inject runtime services (LLMs, storage, auth, MCP) via dependency injection
- **TypeScript Client SDK** — build applications that talk to your agents
- **Server** with built-in runtime services:
  - **LLM proxy** — single API for 15+ providers (OpenAI, Anthropic, watsonx.ai, Bedrock, Ollama)
  - **MCP connectors** — connect agents to external tools and APIs via the [Model Context Protocol](https://modelcontextprotocol.io/)
  - **PostgreSQL** — agent state, conversation history, and configuration
  - **Vector search (pgvector)** — embeddings and similarity search for RAG
  - **File storage (SeaweedFS)** — S3-compatible upload, download, and storage
  - **Document extraction (Docling)** — text extraction from PDFs, CSVs, and more
  - **Authentication (Keycloak)** — identity and access management
  - **Observability (Phoenix)** — LLM tracing and agent debugging
- **Web UI** — chat interface for interacting with and testing your agents
